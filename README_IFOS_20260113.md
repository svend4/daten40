# daten40 — IFOS (Internet Function OS)

Этот репозиторий ведётся **append-only** (вариант 2): мы **не перезаписываем** существующие файлы, а добавляем новые версии с датой/версией в имени.

## Директории
- `master-index/` — Master Index (версии): основной “позвоночник” проекта
- `blocks/` — блоки IFOS (NEXT-XX и/или IFOS_#####_#####)
- `releases/` — архивы/сборки (ZIP)
- `tools/` — очереди, правила, payloads, инструкции

## Как продолжать работу
1) Добавить новый блок в `blocks/...`
2) Добавить/обновить очередь в `tools/NEXT_QUEUE_*.md` (новым файлом)
3) Выпустить новую версию Master Index в `master-index/`

> В этой стратегии источником истины является **последний Master Index** (по дате/версии в имени файла).
