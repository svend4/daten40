# IFOS 50001–50400 — UI OS (Office‑style Interface for Functions, Bundles, Vitrines) (v1)
Цель: интерфейс “как офисные приложения” для интернета функций:
- режимы работы (как Word/Excel/PowerPoint)
- карточки (как “страницы энциклопедии”)
- мастера (как “пошаговая установка”)
- конференция (как “сводка + сравнение + протокол”)
- журнал действий (как “история изменений”)
Чтобы пользователь не “ищет плагины”, а **работает** с функциями.

Порядок: простое → среднее → сложное.

---

## 50001–50040 — Принцип UI OS: режимы вместо хаоса
Базовая мысль: у пользователя всегда есть **намерение**:
- Читать (Learn/Explore)
- Сравнить (Compare)
- Собрать (Compose)
- Установить (Install)
- Запустить (Run)
- Диагностировать (Troubleshoot)
- Опубликовать (Share/Export)
UI OS делает эти намерения явными кнопками/вкладками.

---

## 50041–50090 — Modes: 7 режимов “как офисный пакет”
### Mode A — Reader (читалка/энциклопедия)
- KBCard viewer, источники, версия, доверие
- быстрые рецепты “сделать похожее”

### Mode B — Compare (портал сравнения)
- сравнение товаров/услуг/плагинов/бандлов
- критерии, веса, объяснение рейтинга

### Mode C — Compose (сборка/конструктор)
- drag‑and‑drop шагов (термины из Dictionary)
- предложение совместимых шагов (composability rules)

### Mode D — Install (мастер установки)
- preflight → secrets → deploy → smoke tests

### Mode E — Run (панель запуска)
- старт/стоп, health, логи, расписание

### Mode F — Troubleshoot (диагностика)
- типовые ошибки → исправления
- сбор диагностики одним кликом

### Mode G — Publish (витрина/экспорт)
- экспорт в zip/compose/helm/make/n8n
- публикация в marketplace

## 50091–50140 — Cards: универсальная карточка (UI‑атом)
Карточка (Card) объединяет:
- Summary + Use cases
- Capabilities (из Registry)
- Trust badges (audited/verified/community)
- Install/Run кнопки
- Examples/Recipes
- Reviews (мнения)
- “Сравнить с…”

Карточка — это “страница Word”, но для функций.

---

## 50141–50190 — Wizards: мастера (одна кнопка без магии)
Мастер установки:
1) выбор target (docker/wp/make/n8n/k8s)
2) preflight (авто‑проверки)
3) ввод secrets (с подсказками)
4) deploy (прогресс + логи)
5) smoke tests (проверка)
6) receipt (отчёт + ссылки + next steps)

Мастер сборки (Compose Wizard):
- “что вы хотите сделать?” → intent
- “где запускать?” → target
- “какие ограничения?” → constraints
- результат: macro spec + bundle manifest

---

## 50191–50240 — Conference Mode: “новостная конференция” как продукт
Конференция = режим, который:
- собирает источники (news/events/reviews)
- удаляет повторы, кластеризует темы
- делает 3 слоя: кратко → средне → подробно
- фиксирует “протокол”: что изменилось, что важно, что делать
Это ваш кейс “новости как конференция” — встроенный режим UI.

---

## 50241–50290 — Action Journal: “история” как в документах
Журнал:
- кто/что/когда изменил
- какие шаги запускались
- какие ошибки были
- какие ключи/настройки изменились (без утечки секретов)
- экспорт в отчёт для команды/клиента
Журнал превращает “хаос сценариев” в управляемый процесс.

---

## 50291–50330 — Quality & Trust Panels: качество, доверие, риски
Панели:
- Trust: audited/verified/community
- Security: secrets, permissions, known CVEs (если есть)
- Quality: документация, тесты, покрытие примерами
- Freshness: когда обновлялось
- Cost: приблизительная стоимость (API, compute)
Пользователь понимает “почему это выше в рейтинге”.

---

## 50331–50370 — Search & Filters: поиск по функциям, а не по названиям
Фильтры:
- по функциям (Dictionary terms)
- по capabilities
- по target (self‑hosted/cloud)
- по license/cost
- по trust level
- по совместимости (WP version, Node version)
Именно это решает проблему “миллионы приложений, но никто не видит”.

---

## 50371–50400 — Navigation + Roles + Accessibility: как это живёт в реальности
- Навигация: Home → Dictionary → Registry → Bundles → Run → Journal
- Роли: viewer/editor/admin/security reviewer
- i18n: ru/de/en + fallback
- accessibility: клавиатура, контраст, screen reader
- telemetry: метрики успеха “одна кнопка” (без слежки за содержимым)
