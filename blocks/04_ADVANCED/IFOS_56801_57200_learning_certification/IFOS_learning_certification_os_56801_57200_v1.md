# IFOS 56801–57200 — Learning & Certification OS (обучение и сертификация) (v1)
Цель: чтобы “миллионы функций” стали **осваиваемыми**. Не просто каталог — а обучение “как офисные приложения”:
курс → практика → экзамен → бейдж → путь роста.  
Это ключ к вашему тезису: *рационализация = отдельная инженерная дисциплина*.

Порядок: простое → среднее → сложное.

---

## 56801–56820 — Минимальный режим обучения (Personal)
Что нужно “с нуля”:
- встроенные подсказки в UI (“что это делает”)
- маленькие уроки по 3–5 минут (“micro-lessons”)
- один демо‑макрос “в 1 клик” (быстро показать ценность)
Формат: **Learn → Do → Done**.

---

## 56821–56870 — Learning Paths (пути обучения)
Путь = последовательность кластеров и макросов:
- Beginner: 5–10 уроков + 3 лабораторки
- Standard: 20–40 уроков + 10 лабораторок + экзамен
- Pro: проекты + ревью + сертификация

Примеры путей:
1) “Automation Starter (Make/n8n)”
2) “News/Events Conference Operator”
3) “Comparison Portal Builder”
4) “WP Stack Operator (cache/security/backup)”
5) “IFOS Admin (enterprise governance)”

---

## 56871–56920 — Lessons (уроки как стандартизированные карточки)
Урок = не статья, а карточка:
- цель
- prerequisites
- шаги (до 7)
- типовые ошибки
- проверка результата (что должно получиться)
Уроки привязаны к:
- cluster_id
- macro_id
- connector_id
И должны быть одинакового качества/стиля.

---

## 56921–56970 — Labs (практика на песочнице)
Лабораторка = “сделай руками” на тестовых данных:
- sandbox аккаунты (демо)
- тестовые API ключи (по возможности)
- mock сервисы, если реальный сервис платный/опасный
Лабы:
- “Собери weekly digest” (RSS→dedup→vault)
- “Собери лидогенерацию” (form→sheet→email)
- “Собери WP стабильность” (cache+security+backup)
Автопроверка:
- проверка артефактов (создан файл/запись/лог)
- проверка событий (run completed)
- проверка политик (PII не утекло)

---

## 56971–57020 — Assessments (тесты и экзамены)
Уровни:
- Quiz (короткие вопросы)
- Practical (сделать workflow)
- Scenario (разбор кейса: что не так и как исправить)
- Capstone (мини‑проект)
Оценивание:
- auto‑grader (по результату и логам)
- human review (для Pro/Enterprise)

---

## 57021–57070 — Badges & Credentials (бейджи)
Бейдж = цифровой сертификат:
- уровень (L1/L2/L3)
- какие кластеры покрывает
- срок действия (если нужно)
- проверяемый ID
Важно:
- бейджи должны быть “прикладными”
- связаны с реальными задачами: “умеет собрать X, поддерживать Y”

---

## 57071–57110 — Skillmap (карта навыков по кластерам)
Skillmap = граф:
- узлы = кластеры
- ребра = prerequisites
- веса = сложность, время, популярность
Показывать пользователю:
- “где ты сейчас”
- “что дальше”
- “какие макросы откроются”
Это превращает хаос инструментов в “учебную карту”.

---

## 57111–57150 — Recommendations (персональное обучение)
Используем Analytics:
- если пользователь часто бросает на шаге 2 → предложить урок именно по этому шагу
- если ошибки auth → урок по OAuth/ключам
- если workflow сложный → предложить упрощение (basic macro)
Режим “coach”:
- 1 совет в день
- без спама
- со ссылкой на лабораторку

---

## 57151–57180 — Enterprise training (для команд)
Для компаний:
- назначение курсов по ролям
- прогресс командой (отдел/проект)
- требования compliance (что кто должен пройти)
- отчёты для руководства (PDF/Docx)
Интеграции:
- SSO (Identity OS)
- HR/LMS (SCORM/xAPI опционально)

---

## 57181–57200 — Quality & Governance обучения
Чтобы не было мусора:
- рубрика качества уроков (шаблон)
- версия урока (как code)
- review/approval перед публикацией
- связь с versions bundles (если обновили коннектор — обнови урок)
Это делает обучение **живым** и устойчивым.

---

## Итог
Learning & Certification OS — это “Office‑образование для интернета”:
- путь → практика → экзамен → бейдж
- привязка к кластерам и макросам
- персональные рекомендации по данным
Так рационализация становится массовой.
