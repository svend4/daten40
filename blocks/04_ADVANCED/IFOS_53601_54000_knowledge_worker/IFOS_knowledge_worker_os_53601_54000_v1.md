# IFOS 53601–54000 — Knowledge Worker OS (Office‑режим: документы, карточки, проекты, версии) (v1)
Цель: чтобы IFOS ощущался как **Word/Excel/Notion**, но для “интернет‑функций”.
Не только “пакеты и процессы”, а:
- проекты (workspaces)
- документы (requirements, specs, SOP, чек‑листы)
- карточки (функции, коннекторы, витрины)
- версии/ветки (как Git, но для людей)
- комментарии/ревью (как Google Docs)
- шаблоны и сниппеты (как макросы)
- экспорт (PDF/DOCX/HTML/ZIP)
- линковка знаний (graph)

Порядок: простое → среднее → сложное.

---

## 53601–53630 — Workspace/Projects: “папки и проекты”
Workspace:
- владелец, участники, роли
- настройки языка/политик
- общие каталоги шаблонов
Project:
- цель (one-liner)
- состав (bundles, processes, docs)
- статус (draft/active/archived)
- roadmap (MVP backlog)
Это даёт порядок вместо “хаоса папок”.

---

## 53631–53680 — Documents: “документы как артефакты”
Типы документов:
- Requirements (что хотим)
- Architecture (как устроено)
- Ops/Runbook (как поддерживать)
- Policy (что запрещено/разрешено)
- How-to (инструкция пользователя)
- Conference Note (режим “новостной конференции”)

Документ = объект с метаданными:
- автор, версия, связи с bundles/processes
- теги и статус
- экспортируемость

---

## 53681–53720 — Cards: “карточки функций”
Карточка — минимальная единица знания:
- название (глагол+объект)
- входы/выходы
- условия (policies)
- стоимость/риски
- кнопки: install / run / test
Карточки есть для:
- bundle card
- connector card
- data entity card
- process step card
Это делает систему “понятной визуально”.

---

## 53721–53770 — Versioning & Branching: версии “как в Git”, но проще
Нужно:
- история изменений (timeline)
- diffs (что поменялось)
- ветки (experiment vs production)
- “merge” через approvals
- “rollback” одной кнопкой

Правило:
- каждое изменение фиксируется как version_id
- каждая публикация связана с версией
Это критично для доверия: “почему сломалось” → видно изменение.

---

## 53771–53810 — Comments & Review: совместная работа
Функции:
- комментарии по тексту
- предложенные правки (suggest mode)
- упоминания @role
- чек‑лист ревью
- итог: approve/reject
Это соединяет Knowledge Worker OS и UI Console OS (Approvals).

---

## 53811–53850 — Templates & Snippets: “макросы текста и процессов”
Шаблон = документ с placeholders:
- {PROJECT_NAME}
- {DATA_SOURCES}
- {POLICY_LEVEL}

Snippet:
- кусок текста или шага процесса
- вставляется кнопкой
Каталог шаблонов:
- минимальные (simple)
- стандартные (medium)
- корпоративные (enterprise)
Шаблоны ускоряют внедрение “в 10 раз”.

---

## 53851–53890 — Search/Tagging: навигация по знаниям
Теги:
- domain: news, portal, reviews, marketplace
- stage: draft, prod
- risk: pii, money, legal
Поиск:
- по тексту
- по типу объекта
- по связям (docs ↔ bundles ↔ runs)
Плюс: сохранённые запросы (saved views).

---

## 53891–53930 — Sharing & Permissions: безопасная совместная работа
Sharing policy:
- кто может видеть
- кто может редактировать
- кто может публиковать
Роли:
- owner
- editor
- reviewer
- operator
- viewer

Экспорт “share pack”:
- один ZIP для партнёра/инвестора
- без секретов, с readme и диаграммами.

---

## 53931–53960 — Automation-from-Docs: “документ → процесс”
Ключевая магия:
- документ требований → Copilot план
- чек‑лист → process steps
- SOP → runbook в Console
И наоборот:
- из run отчёта генерируется postmortem doc
Это делает IFOS “живой системой”, а не статичным архивом.

---

## 53961–53990 — Exports: форматы и пакеты
Экспорт:
- PDF (для печати)
- DOCX (для юристов/офиса)
- HTML (для веба)
- ZIP (проект целиком)
Экспорт должен:
- сохранять версии
- включать index.json (manifest)
- не включать секреты/PII

---

## 53991–54000 — Knowledge Links (Graph): связи как карта
Links:
- doc → bundle
- bundle → connector
- process → run
- run → incident
- doc → policy
Граф даёт:
- “где используется этот блок”
- “что затронет изменение”
- “что сломается если убрать коннектор”

---

## Итог
Knowledge Worker OS превращает IFOS в “офис знаний + функций”:
- документы объясняют
- карточки упрощают
- версии дают доверие
- совместная работа ускоряет внедрение
