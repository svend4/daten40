# IFOS 26801–27200 — Workspace & UI OS (v1)
Этот блок делает IFOS **похожим на “офисный пакет”**, где всё разложено по “рабочим пространствам”, “документам/карточкам”, “макросам” и “визуальным схемам”.  
Цель: чтобы человек не утонул в тысяче функций, а работал **как в Word/Excel/PowerPoint**, только для интернет‑функций.

Ниже — **по порядку, от простого к сложному**.

---

## 26801–26830 — Workspace: рабочее пространство (как “папка проекта”)
Workspace = контейнер, где живут:
- коллекции данных (RSS/источники/документы)
- установленные пакеты (из Marketplace OS)
- схемы потоков (FlowGraph)
- UI‑карточки (UI Card)
- макросы и шаблоны

Ключевые поля:
- workspace_id, name
- purpose (“портал сравнения”, “новостная лента”, “CRM‑мини”)
- режим: personal/team
- политики: safe_mode, citations_required, export_allowed (связь с Governance OS)

---

## 26831–26870 — WorkspaceMember + роли: кто что может
Роли: viewer / editor / admin.
Права:
- просмотр карточек/дашборда
- редактирование потоков
- установка пакетов
- управление секретами/экспортом
- публикация витрин

Зачем: B2B без ролей не работает.

---

## 26871–26920 — UI Card: “офисная карточка функции”
UI Card — главный UI‑атом. Это “плитка”/“документ”:
- заголовок + краткое описание
- кнопка Run / Install / Configure
- входы/выходы (как поля формы)
- статус (готово/ошибка/предупреждение)
- вкладка “История запусков”
- вкладка “Источники/цитаты” (если RAG)

Примеры карточек:
- “Новости недели (RSS → бриф)"
- “Сравнить страховки (шаблон портала сравнения)"
- “Импорт отзывов + дедупликация + рейтинг"

---

## 26921–26960 — UI Components + Layout: библиотека элементов
Компоненты: Input, Select, Table, Chart, Timeline, SourceList, DiffView, LogPanel.
Layout описывает:
- сетку (grid), вкладки, секции
- responsive‑поведение (mobile/desktop)
- “режим презентации” (когда показываем партнёрам)

Цель: чтобы карточки были **собираемы из стандартных блоков**, а не каждый раз “вручную верстать”.

---

## 26961–27030 — Flow Builder: визуальные схемы потоков (как “макросы”)
FlowGraph — это “диаграмма действий”:
- Nodes (узлы): ingest, normalize, dedupe, embed, rag_query, export
- Edges (связи): порядок выполнения
- Triggers: manual/schedule/webhook
- Error handling: retry/fallback/skip

Смысл: вместо “тысячи сценариев” — единый визуальный конструктор и библиотека типовых схем.

---

## 27031–27090 — Macro: библиотека повторяемых действий
Macro — “кусок потока”, который можно вставлять (как макрос в Excel):
- входы: параметры
- выходы: результаты
- контракт: что гарантирует
- совместимость: где работает (runtime)

Примеры макросов:
- “Дедуп RSS по заголовку+дате+url”
- “Собрать топ‑10 и объяснить причины”
- “Сделать CSV‑выгрузку”
- “Сделать карточку‑вину (vitrine) из набора”

---

## 27091–27140 — Templates + Forms: “шаблоны документов”
Template — готовая сборка UI Card + FlowGraph + Forms:
- шаблон портала сравнения
- шаблон новостной ленты
- шаблон “каталог плагинов / marketplace витрина”
- шаблон “личная база знаний (RAG)”

FormSchema описывает, что спрашиваем у пользователя, и валидирует ввод:
- URL, ключи, лимиты, фильтры, язык, режим цитирования.

---

## 27141–27180 — Dashboard: панель “как в офисе”
Dashboard = набор виджетов:
- KPI: сколько источников, сколько импортов, сколько ошибок
- “главные новости” / “главные изменения”
- “последние запуски”
- “какие пакеты часто ломаются”
- “рекомендации: что установить дальше”

---

## 27181–27200 — DemoSession + Onboarding: 1‑клик демо и обучение
DemoSession:
- разворачивает шаблон + демоданные
- проводит “тур” по карточкам
- фиксирует успехи (установлено/запущено/получен результат)

Onboarding делает продукт “понятным за 10 минут”, иначе рынок функций не взлетит.

---

## Мини‑архитектура UI OS
1) Workspace создаёт структуру проекта  
2) Marketplace ставит пакеты  
3) Templates создают карточки + потоки  
4) FlowBuilder позволяет редактировать и расширять  
5) Dashboard показывает “что происходит”  
6) DemoSession помогает быстро понять и показать партнёрам
