# IFOS 28801–29200 — UI/UX & Vitrines OS (v1)
Цель блока: сделать так, чтобы **сотни тысяч функций** (плагины/Make‑сценарии/API‑команды) стали:
- **видимыми** (каталог/панели),
- **понятными** (карточки и сценарии),
- **сравнимыми** (портал сравнения),
- **управляемыми** (кнопки Run/Install/Export),
- **объединяемыми** (витрины задач, “офисные приложения для интернета”).

Ниже — по порядку: от простого к среднему и далее к сложному.

---

## 28801–28830 — UI Card: карточка функции (атом интерфейса)
UI Card — минимальная “табличка понимания”, аналог “кнопки + мини‑инструкции”:
- что делает
- входы (какие данные/секреты/права)
- выходы (какие артефакты)
- кнопки: Run / Install / Configure / Docs
- статусы: ok / degraded / broken (из наблюдаемости)

Карточки стандартизируют описание функций так же, как Word стандартизировал “документ”.

---

## 28831–28870 — Widgets: виджеты для карточек и панелей
Widget — блок отображения/управления:
- status_badge, progress, last_run, logs_link
- input_form (мини‑форма)
- table_preview (5–20 строк)
- chart_preview (если есть метрики)
- compare_row (строка сравнения)
- news_tile (новость/сводка)

Виджетов мало (20–40), но они покрывают 80% интерфейсов.

---

## 28871–28920 — Actions: действия как “UI‑команды”
UI Action — вызов в системе:
- install_package
- run_job
- open_vitrine
- export_bundle
- create_secret_ref
- approve_policy_gate

Действие всегда связано с Governance OS:
- кто имеет право нажать,
- что логируется (audit),
- какие подтверждения нужны.

---

## 28921–28970 — Vitrine: витрина задачи (панель “собери и используй”)
Vitrine — это “мини‑приложение” внутри IFOS:
- цель витрины (например: “Новости дня”, “Сравнить страховки”, “Сбор лидов”)
- набор карточек и виджетов
- один “главный сценарий” + дополнительные опции
- фиксированные фильтры и presets
- объяснение “как пользоваться”

Витрина = заменяет 100 разрозненных плагинов одной понятной панелью.

---

## 28971–29020 — Dashboard: рабочий стол по направлениям
Dashboard агрегирует витрины и ключевые метрики:
- мои витрины
- последние запуски
- предупреждения (security/policy)
- результаты дня (дайджесты, отчёты)
- “быстрые кнопки”

Это слой “как Windows/Android”, но для функций/процессов.

---

## 29021–29080 — Compare Portal: “Check24‑логика” в универсальном виде
Сравнение — это отдельный интерфейс‑тип:
- единые поля сравнения (price, features, rating, docs_quality…)
- фильтры и сортировка
- профили сравнения (consumer vs pro)
- источник данных (marketplace listings + reviews + benchmarks)

CompareView:
- columns (что сравниваем)
- constraints (фильтры)
- scoring model (как считаем “лучше”)
- explain (почему это место #1)

Главное: сравнение должно быть **воспроизводимо** и объяснимо, иначе это “реклама”.

---

## 29081–29140 — News Conference: “новостная конференция” как продукт
NewsConference — формат “всё в одном”:
- сбор новостей из источников
- дедупликация
- группировка по темам
- выделение главного (Top N)
- контекст и причины (почему важно)
- “позиции” разных источников (сравнение нарративов)
- экспорт: PDF/HTML/Telegram

Это превращает ленту в **собрание с повесткой**, а не в шум.

---

## 29141–29200 — Navigation + Personalization + i18n: долгоживущая UX‑система
Чтобы система не развалилась:
- NavigationTree: устойчивые разделы (“Автоматизация”, “Данные”, “Новости”, “Сравнение”, “Маркетплейс”)
- UserPreferences: свои витрины, язык, уровень детализации (simple/advanced), избранное
- i18nBundle: строки интерфейса и локализация карточек/описаний
- Accessibility: контраст, шрифты, управление без мыши, screen reader

Так появляется “офисный стиль”: одинаково понятно везде.
